# Project 5 — Generative AI Applications

A convolutional Variational Autoencoder (VAE) for unconditional image generation,
trained on the `automobile` and `truck` classes of CIFAR-10.

## Submission deliverables

All in [`Generative AI/`](Generative%20AI/):

- **[`generative_model.ipynb`](Generative%20AI/generative_model.ipynb)** — full pipeline: data loading, VAE definition, 20-epoch CPU training, reconstructions, prior samples, latent-space interpolation, qualitative evaluation.
- **[`Generative_AI_Analysis_Report.pdf`](Generative%20AI/Generative_AI_Analysis_Report.pdf)** — 7-section analysis and ethics report (markdown source in [`Generative_AI_Analysis_Report.md`](Generative%20AI/Generative_AI_Analysis_Report.md)).
- **[`requirements.txt`](Generative%20AI/requirements.txt)** — pinned environment generated by `pip freeze`.

Supporting artefacts: `training_history.csv` (per-epoch metrics) and `fig_*.png` (figures embedded in the report).

## How to reproduce

```powershell
# 1. Create a clean Python 3.9+ venv
py -3 -m venv .venv
.\.venv\Scripts\Activate.ps1

# 2. Install pinned dependencies
pip install -r "Generative AI/requirements.txt"

# 3. Execute the notebook end-to-end (CIFAR-10 auto-downloads on first run)
python "Generative AI/scripts/run_notebook.py"
```

On a CPU-only machine training takes roughly 14 minutes. CUDA is not required.
